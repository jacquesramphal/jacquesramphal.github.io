## Continued: Deployment models

Three distinct models explored — they are different products with different buyers and different margins.

### Model 1: Configured tool deployed to a team

Agent embedded in the team's existing workflow (Figma plugin, Slack command, GitHub Action). Validates output against token system and Storybook, produces structured handoff spec. One-time build + retainer. You're the architect and the installer. Low ongoing involvement once running.

**Buyer:** Head of Design, VP Engineering
**Ceiling:** $15–30k build + $2–4k/month retainer

### Model 2: You as fractional operator across multiple clients

The agent runs. You monitor it, handle escalations, tune prompts, make judgment calls it can't. You're billing for availability and expertise, not hours. Three clients at $2–3k/month = $6–9k/month for ~10 hours of active work. The leverage is real: the agent does the execution, you provide the oversight layer.

**Buyer:** Same as Model 1, but retention-focused
**Ceiling:** Higher — scales without proportional time cost
**Key insight:** You're not selling design work. You're selling a running system with a human judgment layer on top.

### Model 3: Agent deployed to businesses without a designer

Different ICP entirely — smaller budget, simpler stack, more hand-holding required. Competing with cheap freelancers and no-code tools. Thin margin, high complexity for a different reason. **Skip for now.**

**Recommendation:** Model 2 is the most scalable. Model 1 is the entry point. Model 3 is a different business.

## Continued: Augment vs. replace

The honest question: is it more valuable to augment an existing design team or replace one?

### The real answer

**Augmenting** is the ethical comfortable answer. Lower-value market. Selling "your designers will be faster" — tooling pitch, soft ROI, lower willingness to pay.

**Replacing** is the uncomfortable answer and the higher-value market. A company paying $90k/year for a designer will pay $3–5k/month for a system that produces equivalent output. $36–60k/year vs. $90k — easy sell to a CFO. Different buyer: CEO, VP Engineering, not the design lead.

**But the highest-value position is neither.** It's the skeleton crew model.

### The skeleton crew model

A company has one designer doing the work of three, or one designer when they need two. The agent doesn't replace that designer — it multiplies them. One designer + the agent = output that used to require three. The designer champions it because it protects their position. The CFO champions it because the math is obvious.

Why this is the best entry point:
- The existing designer is an internal advocate, not an opponent
- The ROI story is concrete and easy to tell
- Someone still needs to oversee the agent — keeps you on retainer
- You're not asking anyone to fire anyone

### On the "traitor" question

Be honest about what you're building and who benefits. 

If you build a product marketed as "skip the designer" — you're contributing to displacement. That's a real choice. 

If you build infrastructure that lets one skilled designer cover more ground — you're doing what designers have always done: finding the leverage point. Every tool shift (Figma over Sketch, components over one-offs, tokens over hardcoded values) made individual designers more productive and reduced headcount needed for equivalent output. This is the same thing at a new scale.

The technology is neutral. The business model and the marketing determine who benefits.

**Practical guidance for first clients:** Target companies with an existing design function that is understaffed or under-resourced. The problem is acute, the budget is real, and you're walking in to help the designer who's drowning — not to replace them. Once the model is proven, you can decide whether to go upstream toward the replacement market.

## Competitive analysis findings

Full analysis in `competitive-analysis.md`. Key points:

**The gap is confirmed.** No product as of April 2026 occupies the position of validating AI-generated design output against a team's token contract and Storybook, with a human in the loop.

**Market positions currently filled:**
1. Generation tools — Claude Design, Figma AI, v0, Bolt (generate output, no enforcement)
2. Context-provision infrastructure — Storybook MCP, Knapsack MCP (tell agents what to use, don't audit output)
3. Design system management pipelines — Supernova, Tokens Studio (manage the source of truth, don't validate what AI produces)

**Position 4 is unoccupied:** post-generation validation + structured handoff spec + human oversight role.

**Closest competitor: Knapsack.** $10M Series A (Oct 2025). Their Intelligent Product Engine uses MCP for governance of AI agent output. Upstream context-provision model, not downstream validation. Composition playground not yet shipped. Enterprise buyer, not mid-market. Watch their Q3 2026 launch.

**Strategic tailwind:** Claude Design launched today and generates components that don't respect token contracts. It creates the exact problem the service solves. The launch makes the argument stronger.

**Stack decided:** Claude API directly for reasoning. n8n at the edges for integrations. No LangChain. Plain markdown rules files. Node/Python scripts for orchestration. JSON Schema for output validation.
