## 2. Scope of Work

### Included

[List specifically what will be delivered. Be granular — vague scope becomes disputed scope.]

Example for Audit:
- Review of Figma file structure, component library, and token implementation
- Review of Storybook instance and current component coverage
- Review of existing handoff process and any automation in place
- Written report including: gap analysis, recommended agent architecture, prioritized next steps
- 1-hour debrief call to walk through findings

Example for Build:
- Token contract integration: encoding [Client]'s token system as agent guardrails
- Storybook connection: agent validation against [Client]'s live component library
- Handoff workflow: structured agent output including component spec, accessibility notes, and Storybook story stub
- Guardrail definitions: documented list of what the agent flags for human review
- Documentation: written guide for the team on how to use and extend the system
- Team walkthrough: 1 x 2-hour session with design lead and front-end engineer
- 30 days post-launch support (email/async, 48-hour response)

### Excluded

[List anything that might be assumed but isn't included.]

Example:
- Redesign or restructuring of [Client]'s existing design system
- Building or migrating [Client]'s Storybook instance
- New token architecture (this engagement assumes an existing token system is in place)
- Ongoing maintenance beyond the 30-day post-launch period (covered by a separate retainer agreement if desired)
- Any work outside the workflows defined in Section 2
