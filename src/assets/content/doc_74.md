# The Guardrail Problem

AI makes us fast. Dangerously fast. A layout that used to take an afternoon collapses into minutes. A color system, a component library, a full page — generated, iterated, discarded, regenerated. The speed feels like freedom until you look up and realize you're lost.

Figma files balloon. Frames stack on frames. What started as a clean grid becomes sediment. Every AI assist leaves a layer. Nothing is wrong, exactly — but nothing is quite *yours*, either. The signal drowns in the velocity.

> Code slows you down the right way. It asks you to mean it.

Working in code is different. The syntax resists you. You have to name things, structure things, decide what a rule is before you can write it. That friction isn't a bug — it's the discipline the design process needs. You can't accidentally leave twelve versions of a button in a stylesheet. The compiler keeps score.

The answer isn't to slow down everywhere. It's to build the constraints that let you run. Design tokens as law. Component rules as constraints. Agents that watch the work and flag the drift — the redundant style, the off-grid frame, the third shade of gray that crept in. Let AI do what it's fast at, and let code enforce what it's precise at.

Speed is fine. Speed inside a system is design.
